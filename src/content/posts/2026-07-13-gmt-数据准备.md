### 背景

General motion tracking需要大量的数据来堆砌起模型的跟踪能力。数据一般从人体动作捕捉-机器人重定向-转为训练框架兼容的数据格式。过程中还会包含一些预处理的手段来过滤不合理的数据。

第一步是如何读取/可视化人体数据。

### 结论

- 常见的人体动作数据文件格式为smplx/bvh，一般可以通过blender软件进行可视化。
- bvh一个文件就完整定义了骨架+动作参数，所以可以直接导入到blender中。OFFSET定义父子关节相对位置，CHANNELS定义旋转姿态关系。
- smplx的完整表述包含2个文件，模型定义文件(骨架/人体模版的空间描述)+运动参数文件（动作参数+体型描述），模型输入动作参数，输出关节位姿+蒙皮包络。
- bvh vs smplx:

|含义|bvh|smplx|
|-|-|-|
|定义人体结构 模型/骨架|HIERARCHY|smplx_models|
|关节之间的相对位置关系|OFFSET|smplx_models + npz.betas|
关节之间的旋转关系|CHANNELS|npz.trans/root_orient & poses|
每一帧动作内容|MOTION|npz: dim0=T，如poses[i,:]|


### bvh
#### bvh文件原理
一个典型的bvh文件结构为
```
HIERARCHY
ROOT Root
{
	OFFSET 0.0 0.0 0.0
	CHANNELS 6 Xposition Yposition Zposition Zrotation Yrotation Xrotation
	JOINT Hips
  {
    OFFSET -0.00455359416082507 101.2815786611181 1.7763568394002505e-15
		CHANNELS 3 Zrotation Yrotation Xrotation
		JOINT Spine1
    {...}
    JOINT LeftLeg
    {...}
    JOINT RightLeg
    {...}
  }
}
MOTION
Frames: 2
Frame Time: 0.041666666666666664
5.8456324 u v w x y z a b c ...
3.1415926 d e f g h i j k l ...
```
- 整个文件描述人体运动信息，包含2个部分
  1. HIERARCHY-骨架结构aramture: 定义了人体关节的树状结构，从ROOT开始作为树根，每个关节用{}框起来表达自身+子关节的内容
  2. MOTION-动作信息
      - 动作帧信息frame：定义了多少动作帧Frames，每一帧的时间间隔Frame Time
      - 动作帧内容：每一行代表按顺序的CHANNELS的一帧数值内容
- 骨架结构-关节如何理解：每个关节作为一个坐标系，子关节的原点是父关节坐标系中的一个坐标点OFFSET，然后再建立坐标系，坐标系旋转由CHANNELS定义。整个人体运动 = 每个父坐标系的CHANNELS旋转，带动子坐标系整体旋转，再加上子坐标系自身旋转，...（类比urdf中的结构，只是revolute改为了球铰链）。
- 每个关节joint{}内的元素
  1. OFFSET: 当前关节的坐标系原点，在当前父关节坐标系下的空间位置xyz。
  2. CHANNELS: 当前关节的坐标系xyz轴，相对于父关节坐标系xyz轴的旋转方向。=Z / Y / X rotation.
  3. JOINT: 下一个子关节的名称，接{}.
  4. ROOT特殊，CHANNELS不仅有旋转，还有X/Y/Z position,因为父坐标系是世界坐标系，而root需要在世界系下有移动。



### smplx
#### smplx原理
smplx文件包括人体模型定义文件和动作参数文件。将动作参数文件的每一帧参数送入人体模型中，就可以得到每一帧的输出运动信息。
- 输入1：人体模型定义文件：可认为是一个模版/前向模型，输入一帧的相关参数，通过模型计算，输出人体的全部运动信息。
- 输入2：动作参数文件：包含人体身形信息betas，root的位姿、人体各个关节的旋转信息poses
- 输出： 运动信息：人体蒙皮包络（网格顶点verties+网格三角面faces） + 各个关节的位姿信息joints。

#### smplx文件详解
理解：

- 参数文件 = 树状结构 = root + 身体各个关节，包括肢体、眼睛、手、下颌。
- 每个关节都是球铰链，功能是3轴自由旋转，使用axis-angle表征，方向代表轴线，模长代表旋转量。
- 虽然有很多poses，但是实际上都是表示旋转rot而非位置pos，类比dof pos。
- 人体运动的最小表征 = root(位置+姿态/旋转) + 每个关节的旋转，就像机器人一样。

### blender
#### blender理解
blender是动画可视化软件。
- 可以导入bvh等格式文件，然后播放动作；
- 可以导入python script，做文件处理/打印/渲染/相机设置等功能。
- 可以改变文件的动作内容，保存为新文件。
- ...

导入bvh文件后的可视化
- 每个关节可视化为1个pose bone的姿态骨骼，也就是一个骨头长条。骨头的头=当前关节坐标系原点的空间位置，骨头的尾=子关节坐标系原点的空间位置。比如root骨头=root到hips的连线，连线由hips OFFSET决定。
- 有多个子关节时：pose bone必须是父关节-子关节的连线，要求子关节的位置唯一。当有多个子关节时，骨头的尾取N个子关节中的一个中间值A，A到3个子关节真正OFFSET为止用黑色虚线连起来。然后再从子关节OFFSET位置处作为新的骨头头，定义子关节的pose bone。所以1个子关节下，pose bones是连起来的；多个子关节下，pose bones不直接连，而是中间有黑色虚线连接起来。

blender没有直接的smplx支持，但是有smplx_blender_add_on可以使用。可视化的一条路径是前向计算人体+提取joints/vertices+保存.json/.obj等格式+导入blender中可视化。


#### blender常用操作
- 安装：忘了，问ai。类似pip install blender。
- 导入bvh文件：import - bvh, 选择armature，scale自己定（一般0.01）
- 修改bvh文件：打开后，左上角pose mode:按N，右边侧边栏中Transform，可以查看/改变 pos /rot。黄色为可修改的自由度。
